§ 05 — Final year project

Project NAIA.

A comprehensive passenger-flow analytics and forecasting system built for multi-line railway networks, developed collaboratively with Hitachi Rail GTS and deployed on their ALVEA platform.

# naia / config.py
 
forecasting_models = [
   "ARIMA", "SARIMAX", "LightGBM",
]
streaming = "Apache Kafka"
storage  = "ClickHouse (OLAP)"
ci_cd   = "Jenkins"
deploy  = "Kubernetes · ALVEA"
improvement = "+20%" # scripting perf
// Key achievements
  • Researched regression models to forecast passenger flow for multi-line networks.
  • Improved scripting algorithm performance by 20% via ClickHouse batch processing with OLAP architecture.
  • Engineered a Kafka pipeline with producer–consumer schema registry for multi-format data ingestion.
  • Deployed to ALVEA platform using Jenkins CI/CD and Kubernetes, with emphasis on scalability.
  • Benchmarked ARIMA, SARIMAX and LightGBM; identified LightGBM as optimal for multi-line patterns.
